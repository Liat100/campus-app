# ✅ Checklist לבדיקה לפני פוש ו-Deploy

## 🔍 בדיקות טכניות

### 1. בדיקת Build (חשוב מאוד!)
```bash
npm run build
```
**מה לבדוק:**
- ✅ Build עובר בהצלחה (ללא שגיאות)
- ✅ אין שגיאות TypeScript
- ✅ אין שגיאות ESLint חמורות
- ✅ כל הקבצים נבנים בהצלחה

**אם יש שגיאות:** תקני אותן לפני הפוש!

---

### 2. בדיקת Linting
```bash
npm run lint
```
**מה לבדוק:**
- ✅ אין שגיאות linting חמורות
- ⚠️ אזהרות קלות אפשר להתעלם מהן (אבל עדיף לתקן)

---

### 3. בדיקת TypeScript
```bash
npx tsc --noEmit
```
**מה לבדוק:**
- ✅ אין שגיאות TypeScript

---

## 🧪 בדיקות פונקציונליות

### 4. בדיקת סנכרון נתונים (כבר בדקת! ✅)
**מה לבדוק:**
- ✅ פתחי את האתר בשני דפדפנים שונים
- ✅ צרי/ערוכי קורס בדפדפן אחד
- ✅ רענני בדפדפן השני
- ✅ השינויים מופיעים? ✅

**אם זה עובד מקומית, זה יעבוד גם ב-Vercel!**

---

### 5. בדיקת כל הפונקציות העיקריות

**בדקי שכל הפונקציות עובדות:**
- [ ] יצירת קורס חדש
- [ ] עריכת קורס קיים
- [ ] שמירת קורס (כפתור "שמור")
- [ ] שמירה אוטומטית (onBlur)
- [ ] מחיקת קורס
- [ ] חיפוש קורסים
- [ ] מיון קורסים
- [ ] פתיחת תיקיית קורס (כפתור "פתח תיקיית קורס")
- [ ] כל השדות נשמרים כראוי

---

### 6. בדיקת Validation

**בדקי שהשדות החובה עובדים:**
- [ ] יצירת קורס ללא שם - האם מופיעה שגיאה?
- [ ] שמירת קורס עם שדות חובה חסרים - האם מופיעה רשימת חסרים?
- [ ] האם הכוכביות מופיעות ליד שדות חובה?

---

### 7. בדיקת UI/UX

**בדקי שהעיצוב תקין:**
- [ ] כל הכפתורים נראים טוב
- [ ] הטקסטים מופיעים כראוי (RTL)
- [ ] אין בעיות responsive (אם יש)
- [ ] הצבעים והריווחים תקינים

---

## 🔐 בדיקות לפני Deploy ל-Vercel

### 8. בדיקת משתני סביבה (אם יש)

**אם יש לך `.env.local` או `.env`:**
- [ ] וודאי שהקבצים האלה ב-`.gitignore` (לא נדחפים ל-GitHub)
- [ ] ב-Vercel, הוסיפי את משתני הסביבה דרך Dashboard

---

### 9. בדיקת Git Status

```bash
git status
```

**מה לבדוק:**
- [ ] הקבצים שאת רוצה לדחוף מופיעים
- [ ] אין קבצים רגישים (`.env`, `node_modules`, וכו')
- [ ] כל השינויים שלך staged (אם צריך)

---

### 10. בדיקת KV Store (אחרי Deploy)

**אחרי שהאתר פרוס ב-Vercel:**
- [ ] הוסיפי KV Store דרך Vercel Dashboard
- [ ] וודאי שמשתני הסביבה מוגדרים (Vercel אמור להוסיף אותם אוטומטית)
- [ ] בדקי ש-`/api/db-test` מחזיר `success: true`
- [ ] בדקי ששני משתמשים רואים את אותם נתונים

---

## 📋 Checklist מהיר לפני Push

לפני שאת דוחפת ל-GitHub:

- [ ] `npm run build` עובר בהצלחה
- [ ] `npm run lint` עובר (או לפחות אין שגיאות חמורות)
- [ ] בדקת שהנתונים מסתנכרנים בין דפדפנים
- [ ] כל הפונקציות עובדות
- [ ] אין קבצים רגישים ב-commit
- [ ] `git status` נראה תקין

---

## 🚀 אחרי Push ו-Deploy

**בדקי:**
- [ ] האתר עולה ב-Vercel ללא שגיאות
- [ ] KV Store מחובר
- [ ] `/api/db-test` עובד
- [ ] הנתונים מסתנכרנים בין משתמשים שונים

---

## ⚠️ אם משהו לא עובד

**בדקי:**
1. Console Logs בדפדפן (F12 → Console)
2. Function Logs ב-Vercel Dashboard
3. האם משתני הסביבה מוגדרים נכון
4. האם KV Store מחובר לפרויקט
